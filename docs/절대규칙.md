# 절대규칙

이 문서는 `new_youtube_downloader_gpt`에서 작업할 때 가장 먼저 읽는 운영 규칙입니다.

기준 운영 방식은 `D:\AI\coding\Playground\prompt-studio-refactor-01`의 백업/문서/업로드 루틴을 참고하되, 이 프로젝트는 Python GUI + 패키징 결과물이 함께 있는 다운로드 앱이라는 점을 우선합니다.

## 1. 현재 프로젝트 기준

- 참고 프로젝트: `D:\AI\coding\Playground\prompt-studio-refactor-01`
- 현재 프로젝트: `D:\AI\coding\Playground\new_youtube_downloader_gpt`
- 프로젝트 성격: Python 기반 YouTube downloader 데스크톱 앱
- 대표 소스: `main.py`, `gui.py`, `downloader.py`
- 빌드 관련: `main.spec`, `build/`, `dist/`, `build_versioned.ps1`

## 2. 시작 루틴

새 작업을 시작하면 아래 순서를 먼저 지킵니다.

1. `다운로드`
2. `메시지 확인`
3. `백업생성`
4. `데일리 업무 리스트 작성 여부 결정`

최소 기준:

- `backup/YYMMDD` 폴더를 현재 프로젝트 안에 만듭니다
- `backup_note_YYMMDD.txt` 를 남깁니다
- 배포/실행 관련 변경은 오늘 날짜 백업 폴더 기준으로 추적합니다

## 3. 읽는 순서

아래 순서를 그대로 지킵니다.

1. `docs/절대규칙.md`
2. `docs/PROJECT_READ_FIRST.md` 가 있으면 그 문서
3. 없으면 `main.py`, `gui.py`, `README.md` 순으로 확인
4. 오늘 날짜 `backup_note_<date>.txt` 가 있으면 그것
5. 관련 handoff / 작업 메모가 있으면 필요한 것 1개만

## 4. 작업 범위 규칙

- 참고 프로젝트는 규칙만 참고하고 수정하지 않습니다
- 실제 수정은 현재 프로젝트 폴더 안에서만 합니다
- 소스 기준선은 `main.py`, `gui.py`, `downloader.py` 입니다
- `build/` 와 `dist/` 는 산출물로 보고, 의도한 릴리스 작업이 아니면 소스 대신 직접 수정하지 않습니다
- 패키징 변경은 `main.spec` 와 `build_versioned.ps1` 까지 같이 확인합니다

## 5. 문서 closeout 규칙

meaningful update가 생기면 최소 아래를 같은 턴에 정리합니다.

1. `README.md` 또는 실행 가이드 문서
2. 오늘 날짜 `backup_note_<date>.txt`
3. 필요하면 `docs/PROJECT_READ_FIRST.md`

실행 경로가 바뀌었는데 문서가 그대로면 closeout 미완료로 봅니다.

## 6. 업로드 규칙

`업로드`라고 하면 아래를 먼저 확인합니다.

1. local `HEAD`
2. `origin/main`
3. worktree 변경 범위
4. 소스/산출물 구분이 맞는지
5. 오늘 backup / note 정리 여부

Git 상태와 배포 범위를 감으로 설명하지 않습니다.

## 7. 프로젝트 특화 규칙

- 소스 수정과 빌드 산출물 갱신을 섞어 설명하지 않습니다
- 배포용 exe 또는 `dist/` 산출물은 필요할 때만 포함합니다
- `requirements.txt` 변경이 있으면 실행/빌드 영향도 같이 적습니다
- 버전 변경이 있으면 `VERSION` 과 빌드 스크립트 반영 여부를 같이 확인합니다

## 8. 한 줄 명령

`절대규칙 먼저 읽고, new_youtube_downloader_gpt 안에서만 백업을 만든 뒤, Python 소스와 패키징 산출물을 분리해서 다루며 실행 경로까지 같이 정리합니다`
