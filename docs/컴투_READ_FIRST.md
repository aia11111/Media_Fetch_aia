# 컴투 READ FIRST

이 문서는 `컴투 <-> 컴원` 커뮤니케이션 규칙 중 현재 역할이 `컴투`일 때 짧게 읽는 문서입니다.

현재 진실은 항상 `PROJECT_READ_FIRST.md` 부터 봅니다.
부트는 먼저 `절대규칙.md` 를 읽고 시작합니다.

## FIRST READ ORDER

1. `절대규칙.md`
2. `PROJECT_READ_FIRST.md`
3. `컴투_READ_FIRST.md`
4. 오늘 날짜 `backup_note_<date>.txt`
5. `컴원_TO_컴투_HANDOFF_LOG.md`
6. `README.md`
7. `BUILD_AND_RELEASE.md`

## CORE RULE

- 이 프로젝트는 소스와 build/dist 판단을 분리해서 읽습니다
- 컴투가 확인한 내용은 채팅에만 남기지 않고 문서로 write-back 합니다
- 최소 `PROJECT_READ_FIRST.md`, `컴투_TO_컴원_HANDOFF_LOG.md`, 오늘 날짜 `backup_note_<date>.txt` 중 필요한 곳에 남깁니다

## DIRECTION RULE

- `컴원_TO_컴투_HANDOFF_LOG.md`
  - inbound
  - 컴투가 읽는 문서
- `컴투_TO_컴원_HANDOFF_LOG.md`
  - outbound
  - 컴투가 쓰는 문서

## EXACT COMMAND WORDS

### `메시지 확인`

- `절대규칙.md` 먼저 읽기
- 그 다음 `PROJECT_READ_FIRST.md`
- 그 다음 `컴투_READ_FIRST.md`
- 그 다음 `컴원_TO_컴투_HANDOFF_LOG.md`
- 그 다음 `README.md`, `BUILD_AND_RELEASE.md`
- 아래 5개 포함하기
  - 무슨 뜻인지
  - 이미 끝난 것
  - 아직 남은 것
  - 다음 첫 액션
  - 현재 worktree 상태
